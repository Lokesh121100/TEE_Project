# SC-001 — Password Reset & Account Unlock
## ARIA AI Service Desk | IntelSoft TEE Demo

---

## 1. Overview

SC-001 demonstrates ARIA's ability to handle the most common IT service desk request — password resets — in a **fully automated, end-to-end** manner using real enterprise systems.

| Item | Detail |
|---|---|
| **Scenario** | User's password has expired / account locked |
| **AI Agent** | ARIA (Automated Resolution & Intelligence Agent) |
| **Identity System** | Microsoft Entra ID (Azure AD) via Graph API |
| **Ticketing System** | ServiceNow (custom TEE table) |
| **Chat Interface** | `http://localhost:8000/chat` |
| **Outcome** | Real Microsoft account password reset — not simulated |

## 7. What Makes This Real (Not Simulated)

| Step | What Actually Happens |
|---|---|
| User lookup | Graph API call to `graph.microsoft.com/v1.0/users/{email}` |
| Password reset | `PATCH /users/{id}` with new `passwordProfile` via Graph API |
| Ticket creation | REST POST to ServiceNow `x_1941577_tee_se_0_ai_incident_demo` table |
| OTP | Randomly generated 6-digit code (in production: sent via SMS/email) |
| Temp password | Randomly generated `ARIA@xxxxxxxx1!`, immediately set on Entra ID |

## 8. Troubleshooting

| Issue | Fix |
|---|---|
| Server not starting | Run `python src/api/app.py` from project root |
| "Error connecting to ARIA" | Check server is running at port 8000 |
| User not found | Verify email is `lokesh.jayasankar@intelsoft379.onmicrosoft.com` exactly |
| Microsoft login fails | Run a new chat — old passwords are invalidated after each reset |
| Graph API error | Check app secret hasn't expired in Azure Portal |
| Ollama offline | Demo cache kicks in automatically — flow still works |
